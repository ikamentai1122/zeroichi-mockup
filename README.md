# ボスマッチ — ピッチ用モック

ゼロイチ #10「白→青転職」の **検証用アーティファクト**。動く MVP ではなく、
事業者に見せて「これに出したい/払う」という課金の確約(LOI・デポジット)を取るための
**サイトのガワ(クリック可能モック)**。アプリではない。

- **サービス名**: ボスマッチ(= 「どのボス=実際に教える先輩の下で働くか」を相性でマッチ。boss + match) / タグライン「現場の先輩と、まず一日。」
- **確定プロダクト**: 求職者 × 実際の指導者(現場の先輩)のマッチング + 一日職場体験
- 背景・意思決定は wiki: `wiki/projects/zeroichi/zeroichi-white-to-blue-research.md` 冒頭の
  「🎯 プロダクト確定」「🧪 検証アプローチ確定」/ 入口は `zeroichi-index.md`

## 開き方

依存なしの静的 HTML/CSS/JS。ブラウザで `index.html` を開くだけ。

```
zeroichi-mockup/
├── index.html     事業者向け LP(課題→仕組み→料金→申込フォーム)= 商談の入口
├── register.html  先輩を登録(商談でその場で自社の先輩を登録 → 求職者画面に即反映)
├── seekers.html   求職者の画面(先輩が並ぶ → プロフィール → ライク送信)
├── employer.html  事業者ダッシュボード(届いたライク → 飯(無料) → 有給一日体験(課金¥3万))
├── styles.css     共通スタイル
└── app.js         共通データ(先輩4名・事業者1社)+ ライク/カスタム先輩のロジック
```

## デモの肝(商談での見せ方)

1. **index.html** で価値を説明 — 「採用の意思決定層と現場の先輩のズレ」「見て覚えろで辞める」
   「相性で出会い、一日体験で確かめる」。料金とフォームで **WTP(いくらなら出すか)** を質問。
2. **seekers.html** に切り替え — 「これが求職者から見える御社のページです」。先輩のプロフィールを
   見せ、その場で **ライクを送る**。
3. **employer.html** に切り替え — 送ったライクが **御社のダッシュボードに届いている** のを見せる。
   「まず飯に誘って(無料)、相性が良ければ**有給の一日体験**を設定します」。
   - 「有給一日体験を設定」を押すと **体験コーディネート料 ¥30,000** が明示される
     = ここが課金ポイント。商談の WTP の会話(「これにいくらなら出せますか」)に繋げる。
   - ライクは `localStorage` 経由で seekers → employer に渡るので、商談の場で生きて動く。
   - 右上「デモをリセット」で初期化。

### 強いデモの一手:その場で先輩を登録
商談中に **register.html** で相手の会社の先輩を1人入力 → **seekers.html** に切り替えると
その先輩が並んでいる。「これが御社のページです」と自分ごと化させると刺さる。

## 差し替えポイント(本番化・カスタムデモ時)

- サービス名/タグライン: 各 HTML の `.brand` と `<title>`
- 事業者・先輩データ: `app.js` の `BIZ` / `MENTORS`(商談相手の会社に合わせて差し替えると刺さる)
- 料金: `index.html` の `#price`(現在は仮置き。WTP を測るための数字)

## 既知の割り切り(モックなので)

- フォーム送信・メッセージ送信は実際には飛ばない(画面遷移のみ)。
- データはブラウザの localStorage に保存。サーバ・ログインなし。
- 料金・先輩プロフィールはサンプル。
